Module 1 — Networking Basics · Lesson 1

What is a Computer Network?

A computer network is a collection of interconnected devices that can communicate and share resources with each other.

Why Do We Need Networks?

  • Resource sharing — multiple devices can share one printer or internet connection
  • Communication — email, messaging, and video calls depend on networks
  • Data sharing — files and databases can be accessed from anywhere
  • Centralized management — IT teams can manage all devices from one place

How Data Travels: Packets

Data is broken into small chunks called packets. Each packet travels independently and is reassembled at the destination — this is called packet switching.

🌐 The Internet is the world's largest network — a network of networks all communicating using TCP/IP protocols.
Module 1 — Networking Basics · Lesson 2

Types of Networks (LAN / WAN)

LAN — Local Area Network

Covers a small geographic area — a home, office, or building.

  • High speed (100 Mbps to 10+ Gbps)
  • Low latency — devices are physically close
  • Example: Your home Wi-Fi, an office network

WAN — Wide Area Network

Spans large geographic areas — cities, countries, or the globe.

  • Lower speed than LAN, higher latency
  • Often uses telecom infrastructure
  • Example: The Internet, a company linking branch offices

Other Network Types

  • MAN — Metropolitan Area Network, city-sized
  • PAN — Personal Area Network, very short range (Bluetooth)
  • WLAN — Wireless LAN using Wi-Fi
  • VPN — Virtual Private Network, secure tunnel over public internet
Module 2 — IP Addressing · Lesson 1

IPv4 Basics

An IP address is a unique numerical label assigned to every device on a network — like a postal address for data.

192 . 168 . 1 . 1 ↑ ↑ ↑ ↑ Octet 1 Octet 2 Octet 3 Octet 4 Each octet = 8 bits = values 0 to 255 Full range: 0.0.0.0 to 255.255.255.255

Subnet Mask

A subnet mask divides the IP address into a network portion and a host portion.

IP Address: 192.168.1.25 Subnet Mask: 255.255.255.0 Network: 192.168.1.0 (first 3 octets) Host: .25 (last octet)
💡 IPv4 has ~4.3 billion addresses. We've nearly run out — which is why IPv6 (128-bit) was created.
Module 2 — IP Addressing · Lesson 2

Public vs Private IP Addresses

Private IP Addresses

Used inside a local network. Not routable on the internet.

Private IP Ranges (RFC 1918): 10.0.0.0 – 10.255.255.255 (Class A) 172.16.0.0 – 172.31.255.255 (Class B) 192.168.0.0 – 192.168.255.255 (Class C)

Public IP Addresses

Assigned by your ISP and globally unique. They allow your network to communicate on the internet.

NAT — Network Address Translation

Your router uses NAT to translate between your private internal addresses and your single public IP, letting multiple devices share one public IP.

🏠 Think of your public IP as your street address. Private IPs are room numbers. NAT is the front desk routing mail to the right room.
Module 3 — Networking Devices · Lesson 1

Router

A router forwards data packets between networks — it's the device that connects your local network to the internet.

What a Router Does

  • Connects two or more networks
  • Assigns private IPs to devices via DHCP
  • Performs NAT to share one public IP
  • Maintains a routing table to find the best path for packets
  • Often includes basic firewall functionality
📦 Routers operate at Layer 3 (Network Layer) of the OSI model. They make decisions based on IP addresses.
Module 3 — Networking Devices · Lesson 2

Switch

A switch connects multiple devices within the same local network, allowing them to communicate efficiently.

How a Switch Works

  • Learns the MAC address of every device connected to each port
  • Forwards frames only to the specific port for the destination device
  • Reduces unnecessary traffic (unlike older hubs)

Switch vs Router

  • Switch — connects devices on the same network (Layer 2, MAC addresses)
  • Router — connects different networks (Layer 3, IP addresses)
Module 3 — Networking Devices · Lesson 3

Access Point (AP)

A Wireless Access Point allows wireless devices to connect to a wired network using Wi-Fi.

How It Works

  • The AP connects to the network via Ethernet to a switch or router
  • It broadcasts a Wi-Fi signal (SSID) that devices can join
  • Wireless devices can communicate with wired devices on the same network

Wi-Fi Standards (IEEE 802.11)

  • Wi-Fi 4 (802.11n) — up to 600 Mbps, 2.4 & 5 GHz
  • Wi-Fi 5 (802.11ac) — up to 3.5 Gbps, 5 GHz
  • Wi-Fi 6 (802.11ax) — up to 9.6 Gbps, better in dense environments
