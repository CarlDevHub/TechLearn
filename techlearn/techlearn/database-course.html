Module 1 — Database Concepts · Lesson 1

What is a Database?

A database is a structured collection of information stored electronically so it can be easily accessed, managed, and updated.

Common uses of databases

  • Schools storing student records
  • Hospitals storing patient information
  • Online stores storing products and orders
  • Banks storing financial transactions
💡 Nearly every modern website and application relies on databases.
Module 1 — Database Concepts · Lesson 2

DBMS Overview

A Database Management System (DBMS) is software that allows users and applications to create, manage, and interact with databases.

Functions of a DBMS

  • Stores and organizes data
  • Controls user access
  • Ensures data security
  • Handles multiple users
🔒 Popular DBMS software includes MySQL, PostgreSQL, and Oracle Database.

Tables, Rows, and Columns

Relational databases organize information using tables. Each table contains rows and columns.

  • Column defines the type of data
  • Row stores an individual record
  • Table groups related records

Module 2 — SQL Basics · Lesson 1

SELECT Queries

SQL (Structured Query Language) is used to communicate with relational databases. The SELECT statement retrieves data from tables.

SELECT * FROM students;

The asterisk symbol (*) means all columns will be returned.

SELECT name, grade FROM students;

This query returns only the name and grade columns.

💡 SELECT is the most commonly used SQL command.
Module 2 — SQL Basics · Lesson 2

INSERT Data

The INSERT statement adds new records into a table.

INSERT INTO students (name, grade) VALUES ('John', 'A');

Each INSERT command creates a new row of data inside the table.

💡 Databases can store thousands or even millions of records.
Module 2 — SQL Basics · Lesson 3

UPDATE and DELETE

SQL can modify or remove existing records in a table.

UPDATE example

UPDATE students SET grade = 'A' WHERE id = 2;

DELETE example

DELETE FROM students WHERE id = 2;

The WHERE clause ensures only specific records are affected.

Module 3 — Database Design · Lesson 1

Primary Keys

A primary key uniquely identifies each row in a table.

CREATE TABLE students ( id INT PRIMARY KEY, name VARCHAR(100) );

No two rows can share the same primary key value.

Module 3 — Database Design · Lesson 2

Foreign Keys

Foreign keys create relationships between tables.

CREATE TABLE orders ( order_id INT PRIMARY KEY, student_id INT, FOREIGN KEY (student_id) REFERENCES students(id) );

This ensures the order is linked to a valid student record.

Module 3 — Database Design · Lesson 3

Normalization Basics

Normalization is the process of organizing database tables to reduce redundancy and improve consistency.

  • Eliminates duplicated data
  • Improves database efficiency
  • Makes maintenance easier
💡 Well-structured databases are easier to maintain and scale as applications grow.
